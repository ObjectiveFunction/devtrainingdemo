The limit of one

The single-agent ceiling.

One agent
1

Context overload

One window can only hold so much. A big codebase overwhelms it, and details get lost as the conversation grows.

2

No specialisation

One agent doing the data, the API, the UI and the tests does each one worse. A focused agent writes better code.

3

No coordination

Spawn helpers and they still can't share a task list, talk, or resolve dependencies. More agents, more mess.

This is the limit of one CLAUDE.md in one window — from Session 2.

Keep it trustworthy · 14 min

Quality gates: trust, but verify.

Gate 1

Plan approval

The agent writes a plan; you approve it before any code. Far cheaper to fix a plan than the code.

plan ▸ review ▸ approve / reject ▸ build
Gate 2

Hooks

Automatic checks on events. When a task is "done", the tests run; if they fail, the agent keeps working.

task done ▸ run tests ▸ pass = allow / fail = keep going
Gate 3

AGENTS.md

Write down the patterns and the gotchas. Every agent reads it; every run adds to it.

session 1 learns ▸ AGENTS.md ▸ session 2 avoids it
The real bottleneck
No longer generation.
It's verification.
  • Tests that passed before a change don't prove they'll catch what it breaks.
  • Agents can write tests that look valid but miss the cases that matter.
  • An agent only sees its own window — it misses constraints outside it.

Until checking catches up with making, your review is the safety net.

The honest truth

The human bottleneck was a feature, not a bug.

Work slowly and you feel a mistake early — and fix it. With a team of agents there's no natural brake: small mistakes pile up faster than you can catch them.

what you'd point it at

Goals · get the best out of it

A goal is only as good as its finish line.

Vague goal
"Make the tests pass and improve the code." No clear end — it stops when it looks done, or never stops.
Sharp goal
"All tests in /api pass — npm test exits 0. Don't change any test file." A measurable end it can prove.
Make it provable

The checker reads the conversation, not your disk — so have the agent show the evidence.

Scope and fence it

Name the files in play, and say what must not change.

Keep guardrails on

No test, no goal. Use a branch, iteration and time limits, and review before merge.
